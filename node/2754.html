20gB free cloud space

Submitted by Paul Reed on Sun, 01/09/2013 - 20:02

Some weeks ago, I moved my RootFS on my RaspPi to my trusted Iomega 1 terabit USB HDD because of failing SD cards, and have since used the unused HDD space as a file server and media centre, so all of my music, videos & photos are held there. But then realized that if that drive failed, I've lost the lot, including my emoncms files & backups.

A bit of online searching and I found copy.com which is a online 'cloud' backup service offering 20GB of free 'forever' online storage, owned by Barracuda - the online security & backup experts, and using their own (not rented) servers.

I'm using it now to backup my system - the lot, using their desktop app (gui) but after a bit of digging in their forum, I see that they have just released a client for RaspPi albeit in beta, which will run from the command line, which I'm hoping to integrate and use in preference to the Windows desktop app, especially as uploads are pretty slow compared with local backups. Oh, and they have free mobile apps allowing you to access data on the move.

So why have I posted it in the Sale/Wants/Services forum, well copy.com?r=TYWkfH is an affiliate link which if you signup from that link  then both you and I will get an extra 5GB of free storage making it 20Gb instead of the standard 15GB.
If however you don't want to use the affiliate link (and get 15GB) then just visit copy.com directly.

By way of update, I've now installed the copy.com client on my Raspberry Pi and made a first sync of a portion of my media files, which went well, so I thought I'd document it here in case anyone wants to try it.

First, download the client, and to run the script;

$ cd copy/armv6h

$ sudo ./CopyConsole -u=your_copy.com_email_address -r=/mnt/hdd/folder_that_you_want_to_sync -p=your_copy.com_password

and that's it, the contents of /mnt/hdd/folder_that_you_want_to_sync should start to sync to your copy.com online account.

It's only a beta client at this time, so maybe a little rough around the edges, but between the Barracuda technical team and the clever guys in this forum, I'm sure that it will develop over time.
