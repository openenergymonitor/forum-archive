Submitted by dockarl on Fri, 29/05/2015 - 05:13

Hi Guys -

I've searched high and low to try and find a solution to this but to no avail. I've installed emoncms at emoncms.distantcurve.com. When I create a new user, the write apikey initially shows, in the account page, but if I move off to the inputspage I get the error "Notice: Trying to get property of non-object in /home2/username/public_html/emoncms/Modules/user/user_model.php on line 440". If I then move back to the account page all my details have disappeared and are blank. Login again, same story.

I've tried wiping the install completely, creating a new database, using different versions of emoncms, changing the location of the emoncmsdata folder, and also turning off redis_enabled in the settings file - but always get the same error.

Writing using the apikey also does not work.

I'm using hostgator, on a shared server.

Does anyone have any suggestions? If you'd like to have a go I've left guest registration open for the moment.

Cheers - M

Paul-
 

Thanks for that mate! Yes, you're right, I've resolved the problem over night. For those with similar issues, the fix for me was initially to clear the browser cache and then click the 'remember me' button on the login screen.

I then ran into some further issues, and found that I also had to manually change the version of PHP used in the hostgator control panel for that particular domain to the latest PHP version - so I suspect it was PHP related.
