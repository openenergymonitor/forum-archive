Submitted by TrystanLea on Wed, 11/03/2015 - 08:46.

Hello MikeW, There are two parts to fully fixing this problem, I've done the first part which removes the large spikes caused by a missing day which mostly restores your graphs to a readable form.

The reason for the missing day value is a little more complicated and has been a bug for a while, basically even if there is data in the feed for most of that day, if there happens to be a missing datapoint at midnight written as null to disk it will return null and so appear as if the day is missing even though its only the midnight value.

I've got a fix for this bug working on my test setup which will fill in missing datapoints with the previous valid datapoint giving a fully working wh graph with no missing days. Il be adding this to emoncms and emoncms.org very soon.
