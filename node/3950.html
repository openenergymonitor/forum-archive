Submitted by TrystanLea on Fri, 28/02/2014 - 18:17.

Hello borpin123, this may be an issue associated with the rfm12pi sending time updates combined with old firmware, I had a similar problem which was fixed by updating the firmware. But if you dont need the time sending to an emonglcd you could stop the time sending instead which can be done by either the packetgen module if your using the harddrive image/build or via oem_gateway config or raspberrypi module config.

if your rfm12pi adapter board is version 2, follow the instructions here to update the firmware
http://wiki.openenergymonitor.org/index.php?title=RFM12Pi_V2#Upgrading_R...
