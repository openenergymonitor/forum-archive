Reset or store Zero if feed is inactive for a longer period

Submitted by firefox7518 on Fri, 04/03/2016 - 16:50

Hi

I use EmonCMS for several hundred solar micro inverters of the brand Enecsys. They have a high failure rate and Users actually want to see if an inverter fails.

A micro inverter is actually and will be handled like a full grown normal solar inverter. So we have DC Input side, AC Output and Grid Voltage and Frequency. However I use some bar widgets to show the power output per panel quite a lot for my clients and there is something missing to show that.

So I created a simple AC feed "log to feed" which shows the power output. This feed is used then for the bar widget. When the inverter is not sending any data the last value remains there until the next update. This of course is confusing as you still see values even if there was no update for hours or, in my clients case, days.

The input was showing inactive so it was not sending data for days.

Is there actually a way in EmonCMS to "reset" an input or feed on a daily basis? There are reset to zero processors but I don't think they are doing something like I need it.

With the old event module, it was easy to set up something which at least sends you an e-mail when there was an inactive feed. In newer versions of EmonCMS the event module is not working anymore as we all know.

What do others think about that? Does it not make sense to actually have an option to reset a feed on a schedule or something like this?

Dashboard example: http://swiss-solar-log.ch:85/dliggett The inverter in the top right corner was always showing 170W. We just detected it because I'm in Switzerland and the client in Las Vegas and therefore I was visiting his dashboard during his night time :-)
