I thought I'd installed a local copy of emoncms without pecl/pear, but couldn't remember for sure. (I definitely had not performed an emoncms installation on a VPS) So I set up a VPS (ovh.com) and did not install pear or pecl. All of the other dependencies listed in the docs at https://github.com/emoncms/emoncms/blob/master/docs/LinuxInstall.md (except for ufw) were installed. None of the add-on modules were installed. After copying and modifying settings.php, I was able to register a user and send data to the newly installed emoncms, viewable here: http://167.114.114.94/emoncms/meter

As a kind of double-check, I installed the same configuration on a local linux box running Debian 7.4.0 (Wheezy) and got the same results, i.e a working instance of emoncms without pecl or pear.
