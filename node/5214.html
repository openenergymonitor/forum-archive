Input Process Disable feature.

Submitted by dBC on Sun, 25/05/2014 - 22:48

I'd like to suggest a feature request for consideration in the Input Process mechanism.  As well as having a garbage can in the right hand margin of each process in the chain, it would be great to have an "Enabled" tickbox.  When ticked, the input process would behave as it does now, when unticked it would do nothing other than pass its input value through to the next process in the chain.

At the moment I'm trying to fine tune a calibration constant that is many steps up from the bottom of the chain.  In order not to lose any data as I fine tune, I don't want to kill the feed while I tune it, but it takes me longer than one feed interval to shuffle the new  value into place.    It would be great if I could create it, disable it, arrow it into place at my leisure, and then enable it when ready.
