Lost EmonCMS process list and feedvalue control values

Submitted by pb66 on Thu, 20/02/2014 - 11:26

I am having some trouble with my emoncms.org hosted account.  I have been using it for months without problem, I regularly add or change stuff and everything was ok until yesterday,

I added a couple of inputs, added some processes and started creating a dashboard,nothing unusual until I noticed I was not getting any values in the newly added feedvalue controls, assuming something I had done created the problem I started undoing the changes I had made. whilst removing some processing entries from an input the remaining entries disappeared.

If that wasn't strange enough I can see the results of those "missing" processes in the feed list are still updating as usual, this would suggest they are still there but not visible. The process list for that input is blank, all the others are populated as usual.

I then also noticed it wasn't in fact just my newly added "feedvalue" controls that were missing their values but all feedvalue controls on all dashboards where displaying a value of zero, they had previously worked correctly prior to this point.

As the actual values where all present and correct in the feeds list I went to check to see if the controls configurations had changed but found when I go into a dashboard and enter edit mode as soon as I select a feedvalue control to check all the feedvalue controls on that dashboard populate with correct and updating values but as soon as I click the eye to exit edit mode they return to zero again.

I have waited to see if it sorted itself overnight and it hasn't, I'm really not sure what to try as the underlying data and processes are there, but unaccessible.

Any suggestions ???

Paul

Submitted by pb66 on Fri, 21/02/2014 - 15:39.

So I've eventually got this resolved. 

Top-Tip  - Do not use any symbols in feed names. I found where i had used symbols although emoncms allows it, it doesn't like it, removing all symbols from feed names repopulated all my feedvalue controls.

Never got to the bottom of why all the processes on one input vanished but found it is possible to delete/reset all processes for an input without deleting the input its self so I was able to rebuild the processes and all is well again.
