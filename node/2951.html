Hey Trystan,

 

I have also notice an issue with emoncms, I lost power at my house to day for around 5 hours, before this everything was fine and all feeds were reporting correctly. Two things happen when emoncms does not receive reading from the base unit. 

1. It assumes and average usage base on last reading and the next reading it receives. Now for consumption this might be true however to solar production in my case it can't be. I have a grid-tied setup and if I lose power I don't produce anything.

2. and my kWhd feeds generate stupid readings, I should generally never use more that 15-20kWhd, yet right now it's reporting that I have already used 96kWhd which is impossible. 

 

The issue will all of this basically mean any kind of stats recording is useless and inaccurate. The suggestion I have on resolving this, is if emoncms does not receive a input from the base it should mark it as a 0 until it get's that update, the base unit should be resilient enough that if it can't connect to the emoncms is should buffer the data until such a point as it can.

 

I am using the RPi and oem_gateway to report the emoncms.oeg system.
