Submitted by JBecker on Fri, 28/03/2014 - 08:00

After the many small changes and additions during the last days Emoncms V8.0.6 is running very nicely for me on Windows Home Server. Thank you very much, Trystan!

I am using Firefox (28.0) as default browser and there are two little issues with this browser:

Everywhere where a colour can be chosen (e.g. for the multigraph line colour), only a hash sign # appears. If this is clicked, the text (the hash sign) can be edited and a colour value has to be entered numerically. In Chrome a vertical colour 'bar' appears and if it is clicked, a 'rainbow' colour selection window opens instead. This is much nicer :-)

( in Chrome the colour 'bar' is always black the first time although the line is shown in a different colour in the graph, but this is only a cosmetic problem)

The next issue is the missing feed engines in the dropdown list when choosing a feed engine. All items below the 'other' headline are not shown in Firefox (same in Chrome. Is this because there are no 'other' engines available?)

Another point that is only cosmetic again (?), is that in Firefox a lot of errors are shown in the console window. Most (or all) do not seem to be fatal, but it is a bit strange that there are so many at all.

I am afraid I can not help with clearing these issues, I just try to document them.
