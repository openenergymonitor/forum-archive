monitor my home.  Suggestions to make it even better:

Submitted by Steve on Wed, 17/06/2015 - 08:52.

Adding a "H" option to show 1 hour of data is a one-line change in emoncms on the Pi ....

File /var/www/emoncms/Modules/app/mysolarpv/mysolarpv.html

Insert a line above the "D" option at line 56:

<span class='visnav time' time='0.04167'>H</span> |

The time parameter is the number of days, and 0.04167 is 1/24 which will show 1 hour of data.

Re: Wishlist for Emoncms "My Solar" page

Submitted by Steve on Thu, 02/07/2015 - 19:11.

The improved app works well (I'm using an early Nexus 7 tablet as a permanent display).
More ideas ....

  • POWER NOW is not followed by a colon, therefore EXPORTING should not be.
  • Axis labels are just about readable on a monitor but too small on a tablet.  Can the contrast be turned up (more white, less grey) and the size increased?
  • Suggest the lower labels HOUSE - SOLAR - DIRECT - FROM GRID move to below the numbers.  This will help on landscape tablets where vertical space is tight.
  • When wattages are low, rounding errors come into play like in my screenshot where 1.8 displays as 98% of 1.8.  Currently all numbers are toFixed(1).  Suggest to increase the precision when values are small, aiming for 2 significant digits ....

Add in some common place:

// Like toFixed() but alter precision to show 2 significant digits
function myFixed(n)
{
        var val=Math.abs(n);
        if (val > 99.999) return n.toFixed(0)
        else if (val > 1.999) return n.toFixed(1)
        else if (val > 0.1999) return n.toFixed(2)
        else return n.toFixed(3)
}

Then in mysolarpv.js, replace for example
total_solar_kwh.toFixed(1)
with
myFixed(total_solar_kwh)

 

Steve

Re: Wishlist for Emoncms "My Solar" page

Submitted by Paul Reed on Thu, 16/07/2015 - 10:19.

I made a few changes to make the solar app display better on smaller devices such as smartphones.

  • Removed the colons from 'importing' & 'exporting' labels to be consistent with the other labels
  • Reduced the font size of the power readings slightly so that they don't look cramped (using a CSS rule which only affects small devices such as phones in portrait view - for Tablets & Laptops, & landscape phones the font size will be as before)
  • Reduced the number of selectable time frame buttons so that they don't run into 2 lines (again via CSS, so all of the buttons will be available in landscape view or tablets/laptops etc). The rationale in choosing 1hr, 8hr & Day is that most users would use their phone to check current performance, rather than analyze historical data which would be easier to do on a larger screen such as laptop/tablet. 
  • Changed the label 'FROM GRID' to 'GRID' to ensure alignment with other data.

I have made a pull request for consideration, but if anybody wants, the git clone url is https://github.com/Paul-Reed/app.git

Paul
