Re: Almost blank page when trying to register

Submitted by JD on Sat, 18/10/2014 - 15:56.

I have fought shared hosting restrictions at iPower.com, unsuccessfully I might add.  I moved emoncms to a virtual priviate server ("VPS", such as digitalocean.com or linode.com) and all is well.

However, in my battles with iPower I learned that they don't allow direct user creation of .htaccess files.  When you download the shared hosting emoncms bundle, there is a hidden .htaccess file for emoncms in the emoncms folder.  When you FTP the files  to your hosting account by transferring the whole "emoncms" folder, the hidden .htaccess file is transferred.  

This works fine at GoDaddy.com, but not at iPower.com.  At iPower, you have to go into your control panel and use their .htaccess editor to create and change the contents of .htaccess files.  In my case I had to un-hide the .htaccess file while it was still on my computer, copy the contents of the file, then go online to iPower's .htaccess editor and paste the contents into their .htaccess editor for the emoncms folder.

That got me further than you are :)  I still got stuck at a MySQL permissions problem... but I gave up and went VPS and it works so well there is no going back for me.  Setup time is an extra hour but I have root access to the VPS so I am not at the mercy of some overworked hosting provider support guy.

Re: Almost blank page when trying to register

Submitted by Robert Wall on Sun, 19/10/2014 - 00:44.

The dot in front of htaccess means it's hidden under Linux. Whether you can see it or not in your FTP program depends on which one you use - FireFTP extension for Firefox shows it greyed but accessible. You may need a '-a' option in your FTP program to see it on both the server and locally. The hosting provider I use for websites, but not for emoncms, (Easily.co.uk) doesn't have a restriction on .htaccess, but they allow only one MySQL database and only one user of MySQL - their choice of name for both. So I couldn't have emoncms there if I wanted to, at least not as it's intended to be installed.
"I have root access to the VPS so I am not at the mercy of some overworked hosting provider support guy." - sounds like a big plus!
