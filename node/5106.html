Submitted by Keen2bGreen on Tue, 29/04/2014 - 07:45.

I had a similar problem. If you use the browser function to disoplay the source html code, it continues to reference the old ip address.

I'm assuming your dashboards and required values are all clicked as publec (i.e. little globe, not padlock icons)

I found that if I made the values and dashboard private and then made them public again that it seemed to resolve the issue.  You may also need to delete the charts from you dashboard and re-add them.

Hope this helps
