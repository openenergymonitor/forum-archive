pvoutput.org

Submitted by Lloyd on Sun, 18/11/2012 - 11:24

As well as uploading to my own instance of emoncms, I have also started to upload data to http://www.pvoutput.org.  In my mind it plays a slightly different role to emoncms, allowing you to make comparisons with other systems, and also merge with local weather data. It does have a few quirks as a result of it being based in Australia, but these are minor.  If anyone else is using it, I have created a team (openenergymonitor) for users of components or ideas from this site.

Re: pvoutput.org

Submitted by Lloyd on Sun, 02/12/2012 - 14:17.

I don't use emonbase to send to pvoutput (or my emoncms installation). I had real problems with the reliability of http requests from emonbase, so instead I use my emonbase in server mode, and get another (linux) system to perform http get requests against it, which it then forwards on to emoncms and pvoutput. 

I have a script running on the linux system that polls emonbase for data every 10 seconds.  It then sends this to emonbase using curl, and every 5 minutes it sends a different sequence to pvoutput.org.  For pvoutput, the linux system executes this command:

curl -m30 -d "d=20121202" -d "t=14:07" -d "v2=612" -d "v4=278" -d "v6=235." -H "X-Pvoutput-Apikey:214xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx8" -H "X-Pvoutput-SystemId:1xxx2" http://pvoutput.org/service/r2/addstatus.jsp

where d is date, t time, v2 generating, v4 consuming and v6 voltage.  Take a look here:

http://www.pvoutput.org/help.html#api-addstatus

 

Hope this helps

Lloyd

Submitted by Robert Wall on Sat, 26/10/2013 - 12:16.

MartinR's emonTx PLL monitor records the energy continuously and then sends the result as an average power over the preceding 5 s every 5 seconds. With due attention to the size of the accumulators, there's no reason why that can't be done every 5 minutes (or whatever interval you choose!). There's a full write-up of this project in the pipe-line.

MartinR has also stacked 3 emonTx's for 3 phases.
