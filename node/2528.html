If your feeds are setup as you desire and all you want to do it delete the data collected so far, you can use phpMyAdmin to edit the data in the feed tables directly. It is easy to delete the data while leaving the tables intact.

phpMyAdmin is also useful for backing up and restoring your database, or tables within it, using SQL

Re: How to delete old data?

Submitted by kevinci on Sun, 30/03/2014 - 02:54.

I'm using an install emonCMS v8 on my own webserver.

I've found that by deleting (actually I temporarily moved them to a temp folder) the associated .dat files in the /var/lib/phpfiwa and /var/lib/phpfina folders it resets all old timestore values to 0 upon the next entry into the feed.  I'm assuming the .meta files need to remain to track start and frequency information.

When I return the deleted/moved .dat files back to their original folder (overwriting the new .dat files created), the old data re-appears.

The visualizations reflect the moving of data instantly.
