So, got my monitor back up and running, rasperry pi B+ running emonSD-13-08-14.img with an arduino over USB.  Even found a bad solder joint that meant one of my CT's was centered at 5V at the analog input, not 2.5V.  Interestingly, I had calibrated the error out, and it seemed to work ok even though I was only reading half of a clipped waveform.  Works better now!

I'm having an issue on reboots though - I can see data in the Inputs and Feeds pages, but the feeds don't actually get updated after a reboot.  When I do the following, 

rpi-rw

localemoncms-disable

I get the following error:

Stopping feedwriter:  (feedwriter.php was not running...)

 

Restarting via localemoncms-enable fixes it, but I have to manually run that at each boot.  For power outages, it would be nice if that was automatic.

Any ideas?

Also, I'm finding the raspberry pi is way slower than my old netbook (which was also running off an SD card); it takes around 5 seconds to load a page, and 10 seconds to load a simple multigraph.  Any way to speed that up?  Also, I can't seem to save the Multigraph time axis settings - it always defaults to showing an entire week's worth of data.  I should be able to save changes to that on the Vis Multigraph page, right?

 

Other than that, it's nice to have things working again!  Thanks!  I think I'm going to spring for a wireless emonTx V3 now to monitor an outbuilding.
